NPUB Visualization Approaches — 6 Concepts

Each concept shown for 3 NPUB types: Random | Vanity "meshmate" | Combined
1. ENTROPY WAVE — Sliding window H(bar chart)
Height = local entropy. Low bars = pattern zone. Human spots the "valley" = identity anchor.
Random
All bars tall = no pattern
Vanity Only
Slight dip at start = meshmate
Combined
Deep valley = meshmate in low-ent zone
2. DOMINANT COLOR BADGE — "What color is your NPUB?"
Low-entropy sections produce dominant colors. Random NPUBs = grey/multicolor. Combined = badge with name.
Random
~ no color
Multicolor = grey identity
Vanity Only
meshmate
Same multicolor, name just pasted on
Combined
meshmate 6-color palette
Dominant pink/orange/grey palette = recognizable
3. FINGERPRINT STRIPES — RLE compressed view
Each stripe = a run of same char. Width = run length. Color = char. Like a barcode/fingerprint.
Random (58 chars)
All thin stripes = no runs
Combined
One thick stripe = low-entropy zone with name
Attacker (different pattern)
Different stripe pattern = not Alice
4. PIXEL AVATAR — Entropy-aware identicon
8x8 grid from NPUB data. Symmetric like GitHub identicons. Low entropy = simpler avatar = easier to remember.
Random
Complex = hard to distinguish
Vanity Only
Same complexity as random
Combined
Simple = memorable shape
5. DOT CONSTELLATION — 2D position from char pairs
Each char pair = (x,y) coordinate. Connect dots. Low entropy = tight cluster. Random = scattered.
Random
Scattered everywhere
Combined
Tight cluster = recognizable shape
Attacker
Different cluster location
6. WRAPPED SQUARE — User's idea, refined
58 chars wrapped to 8x8 grid. Low-entropy section = visual region. Size of region proportional to entropy savings.
Random
Full rainbow, no region
Combined
Clear orange region = identity
Attacker
Region in different position
