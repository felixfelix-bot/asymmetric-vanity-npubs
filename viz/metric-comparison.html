Metric Comparison: Recognizability vs Spoofing Prevention

W=16 window · 16 hex symbols (0–f) · Multinomial probability model
Comparing Unique Char Count · Max Frequency · Average Frequency · Combined Score

1 · The Problem with Unique Count Alone

The current primary raindrop metric is unique char count — the number of distinct hex symbols appearing in a window of W characters. Fewer unique = rarer = more recognizable.

The blind spot: Unique count measures diversity but not distribution. Two windows with the same unique count can look completely different:
VERY recognizable
0000000000000abc
unique = 4 · max_freq = 13
Dominated by zeros — eye instantly picks it out.
LESS recognizable
0a0b0c0d0a0b0c0d
unique = 4 · max_freq = 4
Balanced 4-char mix — harder to distinguish at a glance.

Max frequency (count of the most common character) distinguishes these. It captures dominance — how much a single character takes over the window. This is the complementary signal that unique count misses.

3 · Spoofing Prevention — Probability Tables

An attacker generates random keys and checks if they pass the threshold. These tables show the probability per random key of hitting the threshold, and the expected number of keys an attacker must try (1 / P).

3a · P(max_freq ≥ X) — Probability of at least one character appearing X+ times

Computed via multinomial inclusion-exclusion: P(max ≥ X) = 1 − P(all counts < X), where P(all counts < X) = W! · [zW] (Σi=0X−1 zi/i!)S / SW

3b · P(unique ≤ X) — Probability of at most X distinct symbols

Computed via Stirling numbers: P(unique = j) = C(S,j) · S₂(W,j) · j! / SW, then P(unique ≤ X) = Σj=1X P(unique = j).

4 · Combined Score — Capturing Both Dominance and Diversity

The combined score blends max frequency and average frequency (which is W/unique, the inverse of unique count) into a single metric:

combined = max_freq × avg_freq = max_freq × (W / unique)

This product is high when both dominance is strong (max_freq high) and diversity is low (unique small). It penalizes the "balanced few-char" case that unique count alone would over-reward.

WindowUniqueMax FreqAvg FreqCombinedWhat it catches
0000000000000abc 4134.00 52.0 Both low diversity AND high dominance → strongest signal
0a0b0c0d0a0b0c0d 444.00 16.0 Low diversity but NO dominance → combined correctly downgrades
0000000000000000 11616.0 256.0 Maximum signal — single char fills entire window
0123456789abcdef 1611.00 1.0 Random → no signal on any metric
Key insight: The combined score separates "0000000000000abc" (52.0) from "0a0b0c0d0a0b0c0d" (16.0) — a 3.25× difference — even though both have unique=4 and avg_freq=4.0. Unique count alone treats them identically. Max frequency alone sees the difference but ignores diversity. The combined score captures both dimensions.

Verdict: No single metric catches everything. The combined score is the best all-rounder — it detects dominance patterns that unique count misses, while still rewarding low diversity. For maximum security, using both unique count (as a pre-filter) and max frequency (as a dominance check) provides orthogonal coverage. The combined score is a good single-number proxy for both.

6 · Mathematical Details

Definitions

Window: W consecutive hex characters from an npub
Symbols: S = 16 (hex: 0–9, a–f)
unique: Number of distinct symbols in the window
max_freq: Maximum count of any single symbol = max(c₁, c₂, …, cS)
avg_freq: W / unique = average count per present symbol
combined: max_freq × avg_freq = max_freq × (W / unique)

Expected Values (W=16, S=16)

MetricE[X] (random key)Interpretation
unique10.30A random 16-hex window uses ~10 of 16 symbols on average
max_freq3.08The most common symbol appears ~3 times on average
avg_freq1.55W / E[unique] ≈ 16/10.30 ≈ 1.55
combined~4.77E[max_freq] × E[avg_freq] (approximate, not E[product])

Probability Model

Each of the W positions independently draws one of S=16 symbols uniformly at random. The counts (c₁, …, cS) follow a multinomial distribution:

P(c₁=n₁, …, cS=nS) = W! / (n₁! · n₂! · … · nS!) × (1/S)W
where Σ nᵢ = W

P(max_freq ≥ X)

Computed via the complement with exponential generating functions:

P(max_freq < X) = W! × [zW] (Σi=0X−1 zi/i!)S / SW

P(max_freq ≥ X) = 1 − P(max_freq < X)

The polynomial (Σi=0X−1 zi/i!)S is raised to the S-th power, and we extract the coefficient of zW. This is computed via dynamic programming over S bins.

P(unique ≤ X)

Uses Stirling numbers of the second kind, S₂(W,j), which count the ways to partition W items into j non-empty groups:

P(unique = j) = C(S, j) × S₂(W, j) × j! / SW

P(unique ≤ X) = Σj=1X P(unique = j)

Why Max Frequency Adds Value

Unique count answers "how many different symbols?" but not "how are they distributed?". For anti-spoofing, we care about visual recognizability — and recognizability is driven by dominance (one char overwhelming the window), not just diversity.

An attacker who knows we filter on unique ≤ 4 can generate "0a0b0c0d0a0b0c0d" — it passes the filter but is NOT visually memorable. The same attacker cannot easily generate "0000000000000abc" because max_freq=13 is astronomically rarer than unique=4. The combined metric closes this gap.

7 · Tradeoffs & Recommendation

AspectUnique CountMax FrequencyCombined
What it measures Diversity (how many symbols) Dominance (top symbol count) Both
Spoofing resistance Moderate — attacker can use balanced distributions Strong — dominance is harder to fake at high thresholds Strongest — requires both low diversity AND high dominance
Human recognizability correlation Good — fewer symbols = simpler pattern Excellent — dominance is what the eye actually locks onto Best — matches both "few colors" and "one color dominates" intuition
Computational cost O(W) — trivial O(W) — trivial (one pass for frequency map) O(W) — trivial (compute both, multiply)
Sensitivity to window size Scales with W — larger windows naturally have more unique Scales with W — larger windows have higher max_freq Scales with W² — grows faster, needs normalization for different W
False positive rate Higher — flags balanced low-diversity windows that aren't memorable Lower — dominance is a stronger recognizability signal Lowest — requires both conditions
Recommendation: Use max frequency as the primary raindrop metric, with unique count as a secondary signal. The combined score (max_freq × W / unique) is an excellent single-number proxy that captures both dimensions. For the current system, switching from unique-only to combined would immediately downgrade spoofable "balanced few-char" patterns while preserving true high-recognizability windows. The computational cost is identical — one pass through the window.
